ADO Branching Policies in SCB
Branch Types
Protected Branches:
1.	Main - The stable production branch that should always be in a deployable state
2.	Develop - Integration branch where features and bug fixes are merged
3.	Release.* - Prepares code for production deployment, allowing final testing and bug fixes
4.	Hotfix.* - Used for critical fixes in the production environment
5.	Hotfix/ecr/* - Used specifically for Emergency Change Request critical fixes in production
Non-Protected Branches:
1.	Feature.* - Used for developing new features or enhancements
2.	Bugfix.* - Used for addressing bugs found in the development phase

üå≥ Branching Overview

Branching helps developers work on different tasks ‚Äî such as features, fixes, or releases ‚Äî without affecting the main production code.

In SCB, branches are classified into Protected and Non-Protected types.

üîí Protected Branches

These are the core branches of the repository.
They are locked down to prevent direct changes and ensure code stability.

Branch	Purpose
main	Stable production branch ‚Äî always in a deployable state
develop	Integration branch where all features and bug fixes are merged before release
release/*	Used to prepare code for production deployment, final testing, and bug fixes
hotfix/*	Used for critical fixes directly in production
hotfix/ecr/*	Specifically for Emergency Change Request (ECR) fixes in production
üß© Non-Protected Branches

These branches are used for active development and can be created freely by developers.

Branch	Purpose
feature/*	For developing new features or enhancements
bugfix/*	For fixing bugs identified during the development phase

To maintain clarity and traceability, follow standard branch naming formats.

Type	Format	Example
